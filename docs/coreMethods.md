We’re trying to build a **repeatable pricing system for your agency**, the best way to present the research is to break pricing into **pricing models → calculation logic → when they are used → how to turn them into a quick estimation framework**.

Below is a **structured summary of the most widely used pricing methods** by freelancers and tech agencies, based on practices discussed in the fields of Software Project Management, Product Management, and consulting models commonly used by firms like Accenture and Deloitte.

---

# 1. Major Pricing Methods Used by Tech Agencies & Freelancers

These are the **core pricing frameworks** used across the industry.

| Pricing Method            | How It Works                    | Used By                  | Typical Situation           |
| ------------------------- | ------------------------------- | ------------------------ | --------------------------- |
| **Hourly Pricing**        | Client pays per hour of work    | Freelancers, consultants | Unclear scope               |
| **Daily/Weekly Rate**     | Fixed price per day/week        | Consultants              | advisory or contracting     |
| **Fixed Project Pricing** | One price for the whole project | Agencies                 | clear project scope         |
| **Milestone Pricing**     | Project split into phases       | Agencies & startups      | MVP builds                  |
| **Value-Based Pricing**   | Price based on business value   | senior agencies          | revenue-generating products |
| **Cost-Plus Pricing**     | cost + margin                   | early agencies           | predictable profit          |
| **Retainer Pricing**      | recurring monthly payment       | agencies                 | long-term support           |
| **Equity + Reduced Fee**  | cash + startup equity           | startup-focused agencies | risky early startups        |
| **Performance Pricing**   | payment tied to results         | marketing/dev hybrids    | growth platforms            |
| **Feature-Based Pricing** | each feature has a price        | product agencies         | SaaS builds                 |

# 2. The 10 Practical Pricing Methods (Detailed)

## 1️⃣ Hourly Pricing

**Formula**

```
Price = hourly rate × hours worked
```

Typical rates:

| Role       | Freelancers | Agencies  |
| ---------- | ----------- | --------- |
| Designer   | $20–$80     | $60–$150  |
| Developer  | $25–$120    | $80–$200  |
| Consultant | $50–$250    | $150–$400 |

Pros

* simple
* safe for unclear projects

Cons

* client uncertainty
* hard to scale

Freelancers on platforms like Upwork commonly use this.

---

# 2️⃣ Fixed Project Pricing

Most common **agency model**.

**Formula**

```
Estimated hours × hourly rate + risk buffer
```

Example

```
Design 50h
Frontend 120h
Backend 180h
QA 40h
PM 40h

Total = 430h
```

If rate = $40/hr

```
430 × 40 = $17,200
+30% buffer
= $22,360
```

Quote:

```
$22k project
```

---

# 3️⃣ Cost-Plus Pricing

Used by many early agencies.

Formula

```
Total project cost + markup
```

Example

```
Talent cost = $12,000
Overhead = $3,000

Total cost = $15,000
+30% margin

Client price = $19,500
```

Safe for **new agencies**.

---

# 4️⃣ Value-Based Pricing

Price is based on **how valuable the product is to the client**.

Example:

If software helps client generate:

```
$500k yearly revenue
```

Agency might charge:

```
$60k – $120k
```

This model is used heavily by consulting firms like McKinsey & Company.

Pros

* highest profit
* premium positioning

Cons

* requires strong negotiation skills.

---

# 5️⃣ Milestone Pricing

Project broken into phases.

Example:

| Phase       | Payment |
| ----------- | ------- |
| Discovery   | $3k     |
| Design      | $7k     |
| Development | $20k    |
| Launch      | $5k     |

Total

```
$35k
```

Most startup agencies use this.

---

# 6️⃣ Retainer Pricing

Client pays **monthly fee**.

Example

```
$4,000/month
```

Includes:

* feature updates
* support
* improvements

Great for **stable revenue**.

---

# 7️⃣ Feature-Based Pricing

Each feature has a predefined cost.

Example:

| Feature             | Price  |
| ------------------- | ------ |
| User authentication | $1,500 |
| Payment integration | $2,500 |
| Admin dashboard     | $3,000 |
| Messaging system    | $4,000 |

Total price:

```
sum(features)
```

This method is used by many **productized agencies**.

---

# 8️⃣ Tiered Pricing

Offer **packages**.

Example:

| Package       | Price |
| ------------- | ----- |
| Starter MVP   | $10k  |
| Growth MVP    | $25k  |
| Full Platform | $60k  |

Makes selling easier.

---

# 9️⃣ Equity + Reduced Fee

Example:

```
Normal price = $40k

Startup pays:
$15k + 4% equity
```

Risky but sometimes huge upside.

---

# 🔟 Performance-Based Pricing

Example:

```
Base development fee = $10k
+ $2k for every 10k users
```

Often used in **growth partnerships**.
